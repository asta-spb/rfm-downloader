# 3.4.2. Штатные методы — Перечни

**[← К оглавлению](index.md)** · [← Авторизация](03_04_prod_auth.md) · [Штатные ФЭС →](03_04_prod_fes.md)

Все методы: HTTP POST, требуют заголовок `Authorization: bearer <token>`.

---

## Метод 1: suspect-catalogs/current-te21-catalog

**Описание:** Получение информации об актуальном Перечне ТЭ версии 2.1.  
**Запрос:** параметры отсутствуют.

**Ответ:**

| Поле | Тип | Обяз. | Описание |
|------|-----|-------|----------|
| Date | DateTime | О | Дата создания Перечня ТЭ (v2.1) |
| IsActive | Bool | О | Статус Перечня ТЭ (v2.1) |
| IdXml | Guid | О | Идентификатор файла с Перечнем ТЭ (v2.1) |

**Пример:** [3.5.13](03_05_examples_prod.md#3-5-13)

---

## Метод 2: suspect-catalogs/current-te21-file

**Описание:** Получение zip-файла Перечня ТЭ версии 2.1.  
**Content-Type запроса:** `application/x-www-form-urlencoded`  
**Запрос:** `id` — идентификатор из метода `current-te21-catalog`  
**Ответ:** `Content-Type: application/zip` — zip-архив Перечня ТЭ (v2.1).

**Пример:** [3.5.14](03_05_examples_prod.md#3-5-14)

## Метод 7: suspect-catalogs/current-un-file

**Описание:** Получение xml-файла актуальной версии сводного Перечня ООН.  
**Content-Type запроса:** `application/x-www-form-urlencoded`

**Запрос:**

| Поле | Тип | Обяз. | Описание |
|------|-----|-------|----------|
| id | string | О | Идентификатор из метода `current-un-catalog` |
| to_encode | boolean | Н | Признак необходимости кодировки в base64. По умолчанию (`false`) возвращается бинарный массив |

**Ответ:** бинарные данные (xml-файл) Перечня ООН.

**Пример:** [3.5.22](03_05_examples_prod.md#3-5-22)
