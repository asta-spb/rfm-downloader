## 3. Структура наименования, структуры и форматы электронных сообщений

ФЭС формируются в виде файлов формата XML в кодировке UTF-8 (первая строка файла имеет следующий вид: `<?xml version="1.0" encoding="UTF-8"?>`).

### 3.1. Структура наименования информационного ФЭС

Наименование ФЭС имеет следующую структуру:

```
FMTI_V_N_GGGGMMDD_TIXXXPPPPP.xml
```

где:

**FMTI** — идентификатор формы электронного сообщения:
- `FM01` — сообщение об операции с денежными средствами или иным имуществом, подлежащей обязательному контролю, и операции, в отношении которой возникли подозрения об отмывании доходов или финансировании терроризма, а также о приостановленной операции;
- `FM02` — сообщение о принятых мерах по замораживанию (блокированию) денежных средств или иного имущества;
- `FM03` — сообщение о результатах проверки наличия среди своих клиентов организаций и физических лиц, в отношении которых применены меры по замораживанию (блокированию);
- `FM04` — сообщение о фактах препятствия со стороны иностранного государства реализации положений Федерального закона.

В случае представления исправленного или заменяющего сообщения по ранее действовавшим формам используются значения `SPD3` (форма 3-СПД) или `SPD4` (форма 4-СПД).

**V** — код вида предоставляемой информации:
- `1` — операции с денежными средствами или иным имуществом, подлежащих обязательному контролю;
- `2` — операции, в отношении которых возникают подозрения об отмывании доходов или финансировании терроризма;
- `3` — принятые меры по замораживанию (блокированию) денежных средств или иного имущества;
- `4` — результаты проверки клиентов на наличие в перечнях;
- `5` — факты препятствия со стороны государства реализации положений Федерального закона;
- `6` — приостановленные операции с денежными средствами или иным имуществом;
- `7` — почтовые переводы денежных средств с отсутствующими сведениями о плательщике.

**N** — идентификатор отправителя информации:
- для организаций — 19-разрядный код (ИНН + КПП организации (филиала, представительства));
- для лиц установленной категории и ИП — 12-разрядный код ИНН.

**GGGGMMDD** — дата (год, месяц, день) направления электронного сообщения (8 символов).

**TIXXXPPPPP** — порядковый номер электронного сообщения:
- `TI` — идентификатор формы электронного сообщения;
- `XXX` — код филиала организации (присваивается организацией самостоятельно); в ином случае — "000";
- `PPPPP` — порядковый номер в течение дня, начиная с "00001".

Расширение имени файла — `xml` (строчными или прописными буквами).

### 3.2. Структура наименования запроса на удаление ФЭС

```
DFM_N_GGGGMMDD_XXXPPPPP.xml
```

где:
- `DFM` — идентификатор формы запроса на удаление;
- `N` — идентификатор отправителя (ИНН+КПП для организаций, ИНН для ИП);
- `GGGGMMDD` — дата направления;
- `XXX` — код филиала или "000";
- `PPPPP` — порядковый номер, начиная с "00001".

### 3.3. Структура ФЭС

Каждое ФЭС состоит из **служебной** и **информационной** частей.

Служебная часть формируется из показателей таблицы 3.1.

Информационная часть формируется:
- для FM01 — из показателей [таблицы 3.2](rfm261_04b_fm01.md);
- для FM02 — из показателей [таблицы 3.3](rfm261_04c_fm02_fm03.md);
- для FM03 — из показателей [таблицы 3.4](rfm261_04c_fm02_fm03.md);
- для FM04 — из показателей [таблицы 3.5](rfm261_04d_fm04_dfm.md);
- для DFM — из показателей [таблицы 3.6](rfm261_04d_fm04_dfm.md).
